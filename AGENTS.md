# AGENTS.md — HireMe

## Commands

```bash
flutter pub get                                   # Install deps
flutter analyze --no-fatal-infos --no-fatal-warnings  # CI lint
flutter test                                      # All tests
flutter test test/<file>.dart                     # Single test file
flutter test --no-pub                             # CI test (skips pub get)
flutter run                                       # Device/emulator
flutter build apk --release                       # Release APK
firebase deploy --only functions                  # Deploy Cloud Functions (from project root)
```

CI (`.github/workflows/flutter.yml`): `pub get` → `analyze` → `test --no-pub` → `build apk --release` → upload APK → Firebase App Distribution. Flutter `3.41.6`, Java 17.

## Setup

- `.env` at project root (required: `SUPABASE_URL`, `SUPABASE_ANON_KEY`). Listed in `pubspec.yaml` assets.
- `flutterfire configure --project=hireme-a59e6` generates `lib/firebase_options.dart` + `android/app/google-services.json`.
- **Do not edit:** `firebase_options.dart`, `google-services.json`, `.env` (denied in `opencode.json`).
- `pubspec.lock` must be committed (not in `.gitignore`).
- Cloud Functions at `functions/` (Node 20, firebase-functions v5, firebase-admin v12). Deploy from project root.
- `lib/core/utils/app_assets.dart` is auto-generated by `flutter_assets` codegen — do not edit manually.

## Custom OpenCode skills

`hireme_skills/` at root contains 3 repo-specific skills:
- `flutter-getx-controller` — scaffold a GetX Controller + Binding + View. Use **before** writing any new controller/feature screen.
- `flutter-code-review` — self-review checklist. Use **after** completing work, before marking done.
- `flutter-pr` — branch naming, commit messages, and PR description format. Use after code review passes.

## Entrypoint (`lib/main.dart`)

Background isolate handler (`@pragma('vm:entry-point')`) → `dotenv.load` → `Future.wait(Firebase.initializeApp, Supabase.initialize)` with `Firebase.apps.isEmpty` guard → `Get.putAsync(StorageService)` → `Get.putAsync(NotificationService)` → `runApp(GetMaterialApp(initialRoute: Routes.splash))`.

## Architecture

- **State/routing:** GetX. Import only `app/routes/app_pages.dart` — `app_routes.dart` is `part of 'app_pages.dart'`. Use `Routes.*` constants.
- **Modules:** `lib/app/modules/{auth, company, job_seeker, pdf_viewer, profile}`. Auth has sub-directories: `splash`, `onboarding`, `select_user` (active), `role_selector` (unlinked — has bindings/controllers/views but zero `GetPage` entries), `login`, `register`, `forgot_password`.
- **Models:** Inline per-module at `model/` dirs. Also `lib/app/data/models/` and `lib/core/models/` for shared models.
- **Backend:** Firebase (Auth, Firestore, Storage, RTDB, Messaging) + Supabase (`AuthFlowType.pkce`). Firebase project `hireme-a59e6`.
- **Session:** `StorageService` — GetX permanent service backed by SharedPreferences. `AppUserRole` enum (company, jobSeeker). `.value` yields `'company'` / `'jobSeeker'`. `normalizeRole()` maps `jobseeker`/`job seeker`/`job_seeker` → `'jobSeeker'`.
- **Role guard:** `RoleGuardMiddleware` checks `FirebaseAuth.currentUser` + `StorageService`. Mismatch → redirect to other role's wrapper; null → `/login`.
- **FCM tokens** saved to `companies/{uid}` / `jobSeekers/{uid}` (and `users/{uid}` fallback) via `NotificationService._saveTokenToFirestore`. `onTokenRefresh` handles rotation.
- **In-app notifications** in Firestore `notifications/{userId}/items/`. `NotificationService.currentScreen` (`RxString`) suppresses foreground notifications when already on-screen. Both `job_seeker/notifications/` and `company/notifications/` are separate view modules. Dashboard badge streams **must** have `onError` handlers resetting to 0 (subscriptions terminate on error, leaving stale counts).
- **Foreground notification popups** require explicit `flutter_local_notifications` channel (`hireme_notifications`, `Importance.max`). See `NotificationService._handleForegroundMessage`.
- **Cloud Functions** (`functions/index.js`): 4 v2 handlers — `onNewApplication` (Firestore create), `onApplicationAccepted`/`onApplicationRejected` (Firestore update), `onNewChatMessage` (RTDB create). Write notification doc + best-effort FCM push.
- **RTDB chat:** `chats/{chatId}/messages/` with metadata at `chats/{chatId}` (participants, lastMessage, unread counts). Unread: `unreadSeeker` / `unreadCompany` fields, incremented via RTDB transaction, reset on chat detail open. `ChatService.getChats()` queries both `seekerId` and `companyId` simultaneously.
- **Firestore writes:** convention is `SetOptions(merge: true)`.

## Conventions

- Import style: Mixed `package:hire_me/...` and relative — follow file's convention.
- `AppColor` (`app_color.dart`), `CustomTextstyle` (`app_text_style.dart`), `AppString` (`app_string.dart`), `Assets` (`app_assets.dart`, auto-generated) live at `lib/core/utils/`.
- Android-only app (iOS files from Flutter template only).

## Testing

- Manual fakes only — no mockito/mocktail. No Firebase/Supabase init needed.
- Call `Get.reset()` in `tearDown` for GetX isolation.
- 4 test files: `widget_test.dart` (placeholder), `auth_login_controller_test.dart`, `job_seeker_dashboard_controller_test.dart`, `application_review_controller_test.dart`.

## Quirks / Gotchas

- **Broken fonts:** `" Poppins"`, `" Inter"`, `" Segoe.UI"` have leading spaces in `app_text_style.dart` — won't match `Poppins`/`Inter`/`Segoe.UI` declared in `pubspec.yaml`. (`Montserrat` and `Roboto` are correct.) `main.dart` `fontFamily: 'Inter'` (no space) works for the default theme only.
- `Routes.COMPANY_APPLICANTS` defined in `app_routes.dart` but has **no** `GetPage` entry in `app_pages.dart`.
- `Routes.JOB_SEEKER_SEARCH_JOBS` has **no** `RoleGuardMiddleware` — public access.
- `auth/role_selector/` exists (bindings/controllers/views) but zero routes reference it; `select_user/` is the active module.
- **Incomplete logout:** `profile_controller.dart` and `company_profile_controller.dart:logout()` only call `_auth.signOut()` — no FCM token cleanup, no `clearAuthSession()`. Chat-view logout (both `chat_view.dart` and `company_chat_view.dart`) does the full cleanup.
