# AGENTS.md — HireMe

## Commands

```bash
flutter pub get                                   # Install deps
flutter analyze --no-fatal-infos --no-fatal-warnings  # CI lint
flutter test                                      # All tests
flutter test test/<file>.dart                     # Single test file
flutter test --no-pub                             # CI test (skips pub get)
flutter run                                       # Device/emulator
flutter build apk --release                       # Release APK
firebase deploy --only functions                  # Deploy Cloud Functions (from project root)
```

CI (`.github/workflows/flutter.yml`): `pub get` → `analyze` → `test --no-pub` → `build apk --release` → upload APK → Firebase App Distribution. Flutter `3.41.6`, Java 17.

## Setup

- `.env` at project root (required: `SUPABASE_URL`, `SUPABASE_ANON_KEY`). Listed in `pubspec.yaml` assets.
- `flutterfire configure --project=hireme-a59e6` generates `lib/firebase_options.dart` + `android/app/google-services.json`.
- **Do not edit:** `firebase_options.dart`, `google-services.json`, `.env` (denied in `opencode.json`).
- `pubspec.lock` must be committed (not in `.gitignore`).
- Cloud Functions at `functions/` (Node 20, firebase-functions v5, firebase-admin v12). Deploy from project root.
- `lib/core/utils/app_assets.dart` is auto-generated by `flutter_assets` codegen — do not edit manually.

## Entrypoint (`lib/main.dart`)

Background isolate handler (`@pragma('vm:entry-point')`) → `dotenv.load` → `Future.wait(Firebase.initializeApp, Supabase.initialize)` with `Firebase.apps.isEmpty` guard → `Get.putAsync(StorageService)` → `Get.putAsync(NotificationService)` → `runApp(GetMaterialApp(initialRoute: Routes.splash))`.

## Architecture

- **State/routing:** GetX. Import only `app/routes/app_pages.dart` — `app_routes.dart` is `part of 'app_pages.dart'`. Use `Routes.*` constants.
- **Modules:** `lib/app/modules/{auth, company, job_seeker, pdf_viewer, profile}`. Auth has sub-directories: `splash`, `onboarding`, `select_user` (active), `role_selector` (unlinked — has bindings/controllers/views but zero `GetPage` entries), `login`, `register`, `forgot_password`.
- **Models:** Inline per-module at `model/` dirs (dashboard, notifications, profile, etc.). Also `lib/app/data/models/` and `lib/core/models/` for shared models.
- **Backend:** Firebase (Auth, Firestore, Storage, Realtime Database, Messaging) + Supabase (initialized with `AuthFlowType.pkce`). Firebase project: `hireme-a59e6`.
- **Session:** `StorageService` — GetX permanent service backed by SharedPreferences. `AppUserRole` enum (company, jobSeeker). `.value` yields `'company'` / `'jobSeeker'`. `normalizeRole()` maps `jobseeker`/`job seeker`/`job_seeker` → `'jobSeeker'` (camelCase).
- **Role guard:** `RoleGuardMiddleware` checks `FirebaseAuth.currentUser` + `StorageService`. Mismatch → redirect to other role's wrapper; null/unauthed → `/login`.
- **FCM tokens** saved to `companies/{uid}` / `jobSeekers/{uid}` (and `users/{uid}` as fallback) via `NotificationService._saveTokenToFirestore`. `onTokenRefresh` listener handles token rotation.
- **In-app notifications** in Firestore `notifications/{userId}/items/`. `NotificationService.currentScreen` (`RxString`) suppresses foreground notifications when already on-screen. Dashboard header badge uses `JobSeekerDashboardController.notificationBadgeCount` — Firestore stream filtered by `isRead == false`. All badge/count streams **must** have `onError` handlers resetting to 0 (subscriptions terminate on error, leaving stale counts).
- **Foreground notification popups** require explicit `flutter_local_notifications` channel creation with `Importance.max` + `AndroidNotificationChannel`. Handled in `NotificationService._handleForegroundMessage`.
- **Cloud Functions** (`functions/index.js`): 4 v2 handlers — `onNewApplication` (Firestore doc create), `onApplicationAccepted`/`onApplicationRejected` (Firestore doc update), `onNewChatMessage` (RTDB value create). Write notification doc + best-effort FCM push.
- **RTDB chat path:** `chats/{chatId}/messages/` with metadata at `chats/{chatId}` (participants, lastMessage, unread counts). Unread tracking: `unreadSeeker` / `unreadCompany` fields, incremented via RTDB transaction on send, reset to 0 when chat detail opens (`_markChatAsRead`). `ChatService.getChats()` queries both `seekerId` and `companyId` simultaneously. Bottom-nav badge uses wrapper controller's stream from RTDB listening via `.orderByChild(...).equalTo(uid)`.
- **Firestore writes:** always `SetOptions(merge: true)`.

## Conventions

- Import style: Mixed `package:hire_me/...` and relative — follow file's convention.
- `AppColor`, `CustomTextstyle`, `AppString`, `AppAssets` live at `lib/core/utils/`. The sole `app_color.dart` has both the old color group (`kdanger`, `ksuccess`, `kshadow`) and newer colors (`lightThemeGrey`, `lightThemeBlue`, `textPrimary`, `textSecondary`, `divider`, `background`).
- Android-only app (no iOS-specific build scripts; iOS files present from Flutter template).

## Testing

- Manual fakes only — no mockito/mocktail (not in `pubspec.yaml`). No Firebase/Supabase init needed.
- Call `Get.reset()` in `tearDown` for GetX isolation.
- 4 test files: `widget_test.dart` (placeholder), `auth_login_controller_test.dart`, `job_seeker_dashboard_controller_test.dart`, `application_review_controller_test.dart`.

## Quirks / Gotchas

- **Broken fonts:** `" Poppins"`, `" Inter"`, `" Segoe.UI"` have leading spaces in `app_text_style.dart` — won't match declared `Poppins`/`Inter`/`Segoe.UI` in `pubspec.yaml`. (`Montserrat` and `Roboto` are correct.)
- `Routes.COMPANY_APPLICANTS` defined in `app_routes.dart` but has **no** `GetPage` entry in `app_pages.dart`.
- `Routes.JOB_SEEKER_SEARCH_JOBS` has **no** `RoleGuardMiddleware` — public access.
- `auth/role_selector/` exists (bindings/controllers/views) but zero routes reference it; `select_user/` is the active module.
- `lib/app/modules/profile/controllers/profile_controller.dart:logout()` only calls `_auth.signOut()` — no FCM token cleanup (delete from Firestore, `deleteToken()`), no `clearAuthSession()`. Not matching the chat-view logout pattern.
